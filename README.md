
📝 SkillNet 项目需求文档 (Product Requirement Document)
一、 产品概述 (Product Overview)
1.1 项目背景与核心痛点
传统的 Web3 教育与“学习即赚钱 (Learn-to-Earn)”平台面临三大核心挑战：
女巫与脚本套利：自动化脚本和刷课外挂猖獗，恶意抽干项目方的激励资金池。
挂机刷课流：线上技能验证流于形式，无法审计用户是否真正观看了教学视频或仅通过篡改前端数据作弊。
中心化依赖与数据断流：数据高度依赖传统中心化数据库。若使用标准 Solidity 映射（mapping）存储，极易导致前端在高并发拉取商城与排行榜数据时发生断流。
1.2 产品定位
SkillNet 是一个基于以太坊架构构建的 Serverless（无服务器）去中心化学习与技能验证协议。它将“视频学习行为特征采集”与“链下 AI 预言机密码学验签”深度融合，完全移除传统中心化后端数据库，通过全链上智能合约账本实现学习行为的链上存证、激励动态清算以及包含传统商品与 NFT 资产的去中心化商城双重兑换闭环。
二、 核心业务流程与时序 (Business Logic & Sequence)
整个系统的核心业务逻辑分为四个阶段，系统内部不掺杂任何人类情感，只认密码学签名和测试结果：



[ 用户端 DApp ] -- 1. 观看课程视频 + 完成课后答题 --> [ AI 验证节点 ]
      ^                                                            |
      |                                                2. 审计视频停留时长与作弊指纹
      |                                                   绑定最新 Nonce 进行 ECDSA 签名
      |                                                            v
      +----------------- 3. 返回完整 Signature(bytes) --------------+
      |
      |-- 4. 调用 complete_course(带上 video_valid=true) --> [ Solidity 智能合约 ]
                                                                   |
                                                         5. Inline Assembly 解 v,r,s
                                                         6. ecrecover 解析公钥验签
                                                         7. 校验成功: 积分入账 / nonce++
                                                                   |
                                                                   v
                                                         [ 链上原子化清算与资产结算 ]
                                                         - 排行榜动态冒泡排序
                                                         - 商城积分扣除与 NFT 跨合约划转


全量数据拉取：前端用户登录，调用主合约只读接口 get_all_mall_items() 和 get_top_five()，一键拉取商城商品、库存、价格（含 NFT 资产类型）与全局 5 人排行榜数据。
视频学习与 AI 监测：用户在 DApp 端观看课程视频，前端播放器无感采集播放特征（停留时长、倍速特征），连同课后答题数据一并提交给 AI 验证节点。
凭证签发：AI 验证引擎在对答题正确率及视频防刷指纹审计通过后，向合约查询对应用户的最新个人 nonce，将其捆绑打包进行标准以太坊消息签名（signMessage），返回完整 signature(bytes) 给前端。
链上双网关结算：前端调用主合约的 complete_course 方法，提交参数与完整签名。合约内部通过内联汇编（Inline Assembly）动态解出 $v, r, s$，反向解析公钥并与验证者地址比对。验证成功后积分入账、nonce++ 销毁凭证、触发排行榜重排。
资产去中心化兑换：累积的积分可在商城发起兑换。若兑换普通商品，扣除积分并抛出事件；若兑换 NFT，则在扣除积分的同时，原子化触发外部 NFT 接口 safeTransferFrom，直接将标准 ERC-721 荣誉勋章发送至用户小狐狸钱包。
三、 功能模块与详细需求 (Functional Requirements)
3.1 模块 1：智能合约系统 (Smart Contract)
需求 1：EVM 原生 Serverless 账本
用户状态、核心账本与动态计算完全部署于 Solidity 中，优先挂载于以太坊测试网（如 Sepolia）。
需求 2：高安全性链上验签网关
利用内联汇编（Inline Assembly）在链上动态分离提取签名（提取 $r, s, v$），使用 ecrecover 反向解析公钥并与 AI 验证者地址比对，焊死防伪防女巫网络。
需求 3：打破 Mapping 盲盒遍历
在合约底层维护一个活跃的 uint256[] public mall_item_ids 动态数组，暴露全局只读的结构体数组 Getter 接口 get_all_mall_items()，解决以太坊 mapping 无法遍历的痛点。
需求 4：NFT 原子化兑换网关
引入标准 IERC721 接口，重构 redeem_item 逻辑。在扣除用户积分的同时，调用外部 NFT 合约执行资产安全转账，确保资产转移与积分扣除在同一笔交易内完成。
3.2 模块 2：用户端 DApp (Client Frontend)
需求 1：Web3 钱包无缝登录
使用 ethers.js 框架，无缝无感唤起 MetaMask（小狐狸钱包）进行登录与链上交互。
需求 2：智能视频播放器组件
开发专用视频播放组件，实时、无感采集用户的播放行为特征（如停留时长、是否恶意快进特征），作为 AI 预言机防刷审计的数据源。
需求 3：实时事件监听与组件渲染
实现对链上 CourseCompleted、TaskCompleted 以及 NFT 转移事件 Transfer 的实时监听，驱动 3D 跑马灯、全局 5 人排行榜以及用户中心“NFT 荣誉勋章墙”的实时渲染。
3.3 模块 3：管理端前端 (Admin Dashboard)
需求 1：教学资源上架
供课程创作者与系统管理员使用，支持上架教学资源并生成去中心化分发网络内容标识（CID）。
需求 2：多元化商品上架与受控调用
调用合约的管理员受控接口 add_mall_item 往链上录入商城商品。支持录入常规商品，同时支持输入 NFT 的合约地址（nft_address）与对应的 Token ID（nft_token_id），从源头杜绝越权篡改价格漏洞。
3.4 模块 4：AI 验证引擎预言机 (AI Oracle)
需求 1：行为指纹防刷审计
运行轻量级 AI 阅卷与行为监测服务。升级防伪对齐算法，通过时间戳和行为特征审计前端传回的视频防刷、倍速作弊等异常行为。
需求 2：消息绑定与非对称签名
在签发凭证前先向合约查询对应用户的最新个人 nonce，并将其与视频观看记录（video_valid）、答题结果、任务结果捆绑进行标准以太坊消息签名（signMessage），输出单变量完整签名串透传给前端。
四、 非功能性需求与安全防线 (Non-Functional & Security)
4.1 绝对抗女巫与防重放攻击
合约引入基于用户个体的 state.nonce 递增限制机制，AI 引擎生成的每一条签名凭证在链上只能被消费一次。即使黑客拦截了合法的 signature 字节串，由于无法通过 Nonce 校验，也绝无可能进行二次重放攻击。
4.2 三端哈希严格对齐
链下 AI 节点签名时打包的哈希参数顺序，必须与智能合约 keccak256(abi.encodePacked(...)) 内部校验的参数及顺序严格一致（包含新增的 video_valid 字段），否则整个经济结算系统将直接触发 Revert 拦截。
4.3 高扩展性
协议采用极轻量级的纯 Solidity 架构，免去传统中心化服务器和中间件，用户数据和积分资产完全归其私钥控制，可极速平移部署至任意以太坊 L2（如 Arbitrum, Optimism）。
五、 商业化未来规划 (Future Roadmap)
5.1 生产力交付：B端企业级技术悬赏矩阵
协议下一阶段将从单一的学习验证升维为去中心化技术外包与技能雇佣清算协议。
企业悬赏方 (Sponsor) 可直接在 SkillNet 智能合约中锁定 USDT 赏金并发布技术需求与自动化测试用例。
资深架构师 (Architect) 负责构建云端测试用例沙盒与脚手架，按比例抽取持续性知识版税（如 15%）。
极客挑战者 (Challenger) 攻克测试沙盒使代码全绿后，主合约一键触发原子化分账（如 80% 归挑战者，15% 归架构师，5% 平台抽水），并为期铸造不可转让的技能凭证 SBT（灵魂绑定代币）。
